# Requisitos no funcionales

| ID     | Categoría     | Descripción                                                                                |
|--------|---------------|--------------------------------------------------------------------------------------------|
| RNF-01 | Seguridad     | Spring Security, JWT, contraseñas cifradas, autorización por roles, protección de endpoints, Guards en Angular, validación de entradas, no exponer datos sensibles. |
| RNF-02 | Arquitectura  | Hexagonal.                                                                                 |
| RNF-03 | Rendimiento   | Caché para reducir consultas repetitivas de estado.                                        |
| RNF-04 | Consistencia  | Las decisiones administrativas deben ser transaccionales.                                   |
| RNF-05 | Calidad       | Pruebas unitarias con JUnit 5 y Spring Boot Test. Adicionalmente se incluirá SonarQube.    |
| RNF-06 | Persistencia  | PostgreSQL.                                                                                |
| RNF-07 | Usabilidad    | Frontend amigable que permita completar el flujo sin conocimientos técnicos.               |
| RNF-08 | Documentación | Se explicarán y justificarán las decisiones técnicas.                                       |
